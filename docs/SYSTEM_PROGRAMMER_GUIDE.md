# Руководство для системного программиста

## 1. Сведение о программе
Программный модуль для учета заявок на ремонт климатического оборудования.
Заказчик: ООО "Конди".
Исполнитель: Сервисный центр "IT-Сфера".

Система обеспечивает выполнение следующих функций:
- регистрация пользователей в системе;
- создание и хранение заявок на ремонт;
- назначение исполнителей для выполнения работ;
- отслеживание текущего статуса заявок;
- добавление комментариев к заявкам;
- контроль и анализ процесса выполнения работ.

Реализация системы основана на следующих технологиях:
- Backend: ASP.NET Core Web API (.NET 9)
- Frontend: HTML, CSS, JavaScript
- Система управления базами данных: PostgreSQL

## 2. Требования к программным и техническим средствам

### 2.1 Аппаратные требования
Для стабильной работы программного модуля рекомендуется использовать компьютер со следующими параметрами:
- процессор уровня Intel Core i3 / AMD Ryzen 3 или выше;
- оперативная память не менее 4 ГБ;
- свободное дисковое пространство не менее 1 ГБ;
- доступ к локальной сети либо работа через localhost.
При необходимости приложение может быть развернуто на сервере с аналогичными характеристиками.

### 2.2 Программные требования
Для установки и запуска системы необходимо наличие следующего программного обеспечения:
- операционная система Windows 10 или Windows 11;
- установленный .NET SDK версии 9.0;
- PostgreSQL версии 17;
- средство администрирования базы данных (pgAdmin 4 или DBeaver);
- любой современный браузер (Google Chrome, Microsoft Edge, Yandex Browser);
- Git (при работе с репозиторием проекта).

## 3. Работа с приложением

### 3.1 База данных 
SQL-скрипты, необходимые для создания и заполнения базы данных, расположены в каталоге: SQL/
Файлы сохранены в кодировке UTF-8.

### 3.2 Настройка конфигурации приложения
Перед запуском системы требуется настроить файл конфигурации: appsettings.json

В данном файле необходимо указать следующие параметры:
DefaultConnection — строка подключения к базе данных PostgreSQL;
Issuer — издатель JWT-токена;
Audience — получатель токена;
Key — секретный ключ для подписи токена;
ExpireMinutes — время действия токена.

Пример строки подключения:
Host=localhost;
Port=5432;
Database=repair_requests;
Username=postgres;
Password=postgres;

### 3.3 Первоначальный запуск
Перед запуском приложения необходимо убедиться, что установлены все необходимые NuGet-пакеты:
```bash
dotnet add package Microsoft.AspNetCore.Authentication.JwtBearer --version 9.*
dotnet add package Microsoft.AspNetCore.Mvc.NewtonsoftJson --version 9.*
dotnet add package Npgsql.EntityFrameworkCore.PostgreSQL --version 9.*
dotnet add package Microsoft.EntityFrameworkCore.Design --version 9.*
dotnet add package Swashbuckle.AspNetCore.Swagger --version 9.*
dotnet add package Swashbuckle.AspNetCore.SwaggerGen --version 9.*
dotnet add package Swashbuckle.AspNetCore.SwaggerUI --version 9.*
```
После этого необходимо выполнить следующие команды в каталоге проекта:
```bash
dotnet restore
dotnet build
dotnet run
```
После запуска в консоли появится адрес, по которому будет доступно веб-приложение.

### 3.4 Проверка корректности запуска
После старта программы рекомендуется проверить:
- запускается ли сервер без ошибок;
- открывается ли интерфейс Swagger;
- доступна ли клиентская страница в браузере;
- корректно ли работает авторизация пользователей;
- устанавливается ли соединение с базой данных.
Если возникают ошибки, необходимо проверить настройки файла appsettings.json и убедиться, что все зависимости были успешно установлены.

## 4. Структура backend-части проекта
Серверная часть приложения реализована на платформе ASP.NET Core Web API.
Основные элементы проекта:
- Program.cs
Файл точки входа в приложение. В нем выполняется конфигурация сервисов, маршрутизации и middleware.

- Models/
Каталог содержит модели данных, которые соответствуют таблицам базы данных.

- Data/
В данной папке находится контекст базы данных (ApplicationDbContext), где объявляются таблицы и их связи.

- Controllers/
Контроллеры отвечают за обработку HTTP-запросов и реализацию CRUD-операций.

- Constants/
Содержит классы с константами, например перечень ролей пользователей.

- DTOs/
DTO (Data Transfer Object) используются для передачи данных между клиентом и сервером.
Они позволяют ограничить передаваемые поля и повысить стабильность API.

- Services/
В этом каталоге находятся сервисы, отвечающие за безопасность системы:
- хэширование паролей;
- генерация JWT-токенов.
- appsettings.json
Файл конфигурации приложения, в котором задаются параметры подключения к базе данных и другие настройки.

## 5. Структура клиентской части
Файлы пользовательского интерфейса находятся в каталоге wwwroot.
Основные файлы:
- index.html — главная страница приложения.
- login.html — страница входа пользователей в систему.
- site.css — файл со стилями интерфейса.
- app.js — содержит логику работы клиентской части и взаимодействие с API.

## 6. Возможные ошибки при работе системы

### 6.1 Приложение не запускается
Причины могут быть следующими:
- зависимости проекта не установлены (dotnet restore);
- возникли ошибки при сборке проекта;
- неправильно настроен файл конфигурации appsettings.json.

### 6.2 Ошибка соединения с PostgreSQL
Возможные причины:
- сервер PostgreSQL не запущен;
- указана некорректная строка подключения;
- отсутствует созданная база данных.

### 6.3 Ошибка авторизации пользователей
Возможные причины:
- JWT-токен отсутствует или срок его действия истек;
- некорректно настроены параметры Issuer или Audience;
- ошибка при генерации токена.
